# Chapter 8 — Cloud Backup

---

## 1. Cloud Backup Overview

Cloud backup syncs your project data to a remote server, enabling:
- Seamless data recovery when switching devices
- Protection against data loss from device damage or loss
- Data sharing between multiple devices

---

## 2. Enabling Cloud Backup

**Path:** Settings → Backup → Cloud Backup

First-time setup requires:
1. A valid **license** (see Chapter 13)
2. A working internet connection
3. Cloud backup toggle turned on

The Settings page shows the cloud connection status:
- **Connected** (green): backups are working normally
- **Connection failed** (red): check your network or license

---

## 3. Manual Backup

Inside a project: Save/Export menu → **Upload to Cloud**

Or: tap **Backup Now** in the Settings page.

---

## 4. Managing Cloud Backups

**Path:** Settings → Backup → Cloud Backup → Manage

The management page lets you:
- View the list of all backed-up projects
- See backup time and data size for each
- **Download** a specific backup to your device
- **Delete** backups you no longer need

---

## 5. Restoring from Cloud

1. Settings → Cloud Backup → Manage
2. Find the project you want to restore
3. Tap **Download/Restore**
4. Confirm — data imports into your local project list

---

## 6. Notes

- Cloud backup requires an internet connection; use Wi-Fi when possible
- Initial backup of large datasets may take some time
- Cloud storage has capacity limits — periodically clean up old backups
- Cloud backup and local auto-backup (Chapter 9) are independent — enabling both is recommended for double protection
